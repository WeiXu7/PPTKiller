# PPTKiller ToDo

当前 MVP 已完成：用户登录/注册、项目与历史会话、Harness Agent 流程、资料解析、联网资料/文献/图片检索、DeepSeek 大纲生成、演讲稿、专业 PPTX 导出、真实导出缩略图预览。

## P0：把 MVP 做到可真实试用

- [x] Web 端逐页编辑器
  - 编辑标题、要点、讲稿备注、版式类型。
  - 支持逐页保存到 session artifact。
  - 编辑后可重新导出 PPTX。

- [x] 图片替换与分配
  - 每页可选择检索图片、上传图片或不使用图片。
  - 保留图片作者、来源和授权信息。
  - 导出时按页应用用户选择，而不是仅自动轮换。

- [x] 图表/数据页
  - 从 Excel/CSV 抽取表格字段。
  - 支持柱状图、折线图、表格页、关键指标页。
  - 图表页保留数据来源引用。

- [x] 导出 QA 面板
  - 在前端显示每页真实渲染缩略图、导出时间、页数、文件状态。
  - 展示导出警告，例如缺引用、缺图片、图片下载失败、OCR 未执行。
  - 支持下载 PPTX 和查看总览图 montage。

## P1：增强 Agent 与资料理解

- [x] 影像科病例驱动模式：主题与病例准备
  - 支持“只有主题”的入口，先生成病例收集清单，而不是直接生成 PPT。
  - 支持“主题 + 已有病例”的入口，先盘点病例材料、教学价值和缺失信息。
  - 输出病例准备度：可生成、建议补充、不建议生成。
  - 详细产品结论见 `docs/product/radiology-case-driven-mode.md`。

- [ ] 影像科病例驱动模式：病例卡与补齐流程
  - 为每个病例建立结构化病例卡：基本信息、检查信息、影像表现、最终诊断/病理/随访、教学标签、展示方式。
  - 对缺失字段显式标记为待补充，不允许模型编造病例事实。
  - 生成面向医生的补充问题，例如最终诊断、关键图像、检查日期、病理/随访结果、是否已脱敏。

- [ ] 影像科病例驱动模式：教学主线与 PPT 结构
  - 按教学价值编排病例，而不是按上传顺序排列。
  - 支持典型、非典型、鉴别诊断、误诊陷阱、随访/病理验证、多病例对照等页面类型。
  - 生成读片问题、诊断推理链、影像诊断 checklist、报告书写建议和讨论页。

- [ ] OCR / 视觉理解
  - 对扫描 PDF、图片型 PPT、无描述图片按需解析。
  - 避免对有文本层文档重复 OCR。
  - 需要选型：本地 OCR、云 OCR，或多模态模型。

- [x] 文献质量分层
  - 区分学术论文、行业报告、网页资料、新闻。
  - 标记 DOI、年份、作者、期刊/会议、访问链接。
  - 对不可靠来源给出降权或替代建议。

- [ ] Agent 事件流
  - 后端将长任务拆成异步 job。
  - 前端实时展示步骤进度，而不是等待接口返回。
  - 可选 Redis/Celery/RQ 或 FastAPI background task。

- [ ] 会话恢复与版本管理
  - 每次修改保留版本。
  - 支持回滚到某次大纲、某次编辑或某次导出。
